# Daily Activity Logs

본 폴더는 에이전트의 토큰 효율화와 장기적 컨텍스트 유지를 위해 매일의 핵심 작업 내용을 기록하는 공간입니다.

## Logging Rules

1. **파일명**: `YYYY-MM-DD.md` 형식을 사용합니다.
2. **기록 시점**: 일일 작업 세션을 종료하거나 Git 커밋을 하기 직전에 에이전트에게 기록을 요청합니다.
3. **포함 내용**:
   - 주요 변경 사항 (코드, 구조, 정책)
   - 핵심 설계 결정 사항 (Rationale)
   - 미해결 이슈 및 다음 세션에서 진행할 작업
4. **활용**: 에이전트가 매번 모든 프로젝트 히스토리를 읽는 대신, 최근 몇 개의 로그 파일만 읽음으로써 빠르게 컨텍스트를 파악하고 토큰을 절약합니다.
